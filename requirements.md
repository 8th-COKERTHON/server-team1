# 기능 요구사항 명세서 (Requirements Specification)

본 문서는 프로젝트의 기능 요구사항을 정의합니다. 우선순위(Priority)는 1(최상) ~ 4(최하)로 분류됩니다.

## 1. 온보딩 (Onboarding)
| Depth | 기능 | 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| 디지털 디톡스 설정 | 시간 설정 | 시작/종료 시간 설정 및 매일 반복 적용. 설정 필수(홈 진입 조건). | 1 |
| 이름 입력 | 이름 입력 | 사용자 이름 입력. **최대 5자** 제한. | 1 |

## 2. 홈 (Home)
| Depth | 기능 | 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| 디톡스 현황 | 시간 조회 | 설정 시간 확인 및 클릭 시 수정 바텀시트 호출. | 1 |
| 팀 전환 | 팀 선택 | 팀 선택 바텀시트를 통한 팀 변경. 미소속 시 바텀시트 미제공. | 2 |
| 인증 현황 | 팀원 인증 | 전날 미션 성공/실패 여부 시각화(초록/빨강 상태 표시). | 2 |
| 인증 현황 | 개인 인증 | 개인 모드 시 본인 인증 현황 표시. | 2 |
| 목표 | 벽돌 현황 | 오두막집~빌딩 성장형 시각화. 성공(+1)/실패(-1) 반영. | 3 |
| 목표 | 결과 팝업 | 미션 실패 시 실패 안내 팝업 제공. | 3 |
| 팀 관리 | 방 만들기 | 팀 참여 여부와 관계없이 방 생성 이동 버튼 제공. | 2 |
| 알림 설정 | 이메일 등록 | 온보딩 이메일 기반 알림 설정 수정. | 4 |

## 3. 미션 (Mission)
| Depth | 기능 | 설명 | 우선순위 |
| :--- | :--- | :--- | :--- |
| 오늘의 미션 | 미션 팝업 | 시작 시점 랜덤 미션 제공. 10분 타이머 유지. 미수행 시 실패 처리. | 1 |
| 오늘의 미션 | 10분 타이머 | 최대 10분 지연 가능. 앱 종료/미접속 시에도 시간 흐름. | 2 |
| 오늘의 미션 | 랜덤 미션 | 자리에서 일어날 수 있는 랜덤 행동 미션 제공. | 1 |
| 인증 | 사진 인증 | 인증 사진 업로드 및 재촬영 가능. 팀원 전체 공유. | 1 |
| 디톡스 진행 | 진행 화면 | "폰 내려놓을 시간" 문구 및 종료 시간/팀원 현황 표시. | 1 |

## 📌 주요 정책 및 제약 조건 (Policy Constraints)
개발 시 반드시 준수해야 하는 시스템 제약 사항입니다.

1.  **닉네임(이름)**: 최대 5자까지 입력 가능 (`max=5`).
2.  **방(팀) 이름**: 최대 10자까지 작성 가능 (`max=10`).
3.  **벽돌 로직**: 팀/개인 모드 무관하게 전날 미션 성공 시 +1, 실패 시 -1.
4.  **미션 제한**: 하루 1개의 랜덤 미션 제공 및 수행.
